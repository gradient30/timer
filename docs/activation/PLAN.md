# 激活码体系实施计划

## 目标
在不影响现有功能的前提下，完成离线激活并预留服务器激活能力，确保未激活无法使用。

---

## 阶段 A（已完成）：离线激活
### 任务
1. **激活码规范**：16 位分段 `XXXX-XXXX-XXXX-XXXX`
2. **本地校验**：HMAC-SHA256 签名校验
3. **一次性激活（本机）**：激活成功后写入 `config.json`
4. **启动阻断**：未激活仅显示激活弹窗
5. **生成器**：`activation_gen` CLI

### 验收
- 启动必须输入激活码
- 激活成功后功能可用
- 重启应用保持已激活状态

---

## 阶段 B（计划）：服务器激活（延后实现）
### 目标
通过服务端记录激活状态，实现**全局一次性使用**与更强防破解能力。

### 任务
1. **激活接口**：`POST /activate` 记录激活状态
2. **数据存储**：存储激活码使用状态与时间
3. **客户端优先线上**：服务器可用时优先在线校验
4. **离线兜底策略**：可配置（允许或拒绝离线激活）

### 验收
- 码一旦激活无法再次使用
- 换设备/重装无法重复激活

---

## 里程碑
- **A**：离线激活 ✅
- **B**：服务端激活 ⏳
