# 激活码体系说明

## 目标

未激活时禁止使用全部功能；激活码可人工分发，并支持后续接入服务端以实现全局一次性校验。

## 已实现（离线阶段）

| 能力 | 说明 |
|------|------|
| 格式 | `XXXX-XXXX-XXXX-XXXX` |
| 校验 | HMAC-SHA256（编译期注入 `TIMER_ACTIVATION_SECRET_HEX`） |
| 门禁 | 未激活仅显示激活弹窗 |
| 持久化 | `config.json` 记录 `activated`、激活时间、码哈希 |
| 生成 | CLI `activation_gen` / `./scripts/dev.sh activation` |
| 开发后门 | `activation-admin`：托盘提示连点 + 口令，一次生成 5 码（发布包不含） |

## 离线模式限制

- 无法保证「一码仅用一次、全平台通用」
- 同一有效激活码可在多台设备离线激活
- 防伪造依赖签名密钥；密钥泄露需轮换并重新发布

## 计划（服务端阶段）

- `POST /activate` 登记激活码使用状态
- 已使用则拒绝再次激活
- 客户端优先联网校验，失败时可配置是否允许离线兜底
