# 激活码体系需求说明

## 背景与目标
为应用增加激活码体系，未激活时**绝对禁止使用**。支持后续引入服务器验证以实现全局一次性使用与防破解增强。

## 总体方案（原始目标）
- **优先服务器校验 + 离线兜底**
- 当服务器可用时：服务端记录激活码使用状态，确保一次性使用
- 当服务器不可用时：离线校验仍可激活，但无法保证跨设备/重装唯一性

## 当前阶段（已实现）
- **仅离线校验**：
  - 激活码格式：`XXXX-XXXX-XXXX-XXXX`（16 位分段）
  - 本地验证：HMAC-SHA256 签名校验
  - 激活记录：`config.json` 写入 `activated=true`、激活时间与码哈希
  - 入口阻断：未激活时仅显示激活弹窗，其他功能不可用
  - 生成方式：提供本地生成器 `activation_gen`

## 后续阶段（计划实现）
- **服务器激活**：
  - `POST /activate` 接口校验并登记激活码使用状态
  - 若已使用则拒绝再次激活
  - 客户端优先请求服务器，失败时可选是否允许离线兜底

## 核心需求清单
1. 未激活禁止使用全部功能
2. 激活码一次性使用（服务器阶段强制）
3. 激活码不可伪造（签名校验）
4. 激活码格式统一、可人工分发
5. 提供生成器与验收流程

## 约束与风险
- 离线模式无法保证“跨设备/重装”全局唯一使用
- 若要达到**绝对一次性**，必须接入服务器并记录激活状态
