> **版本**: v1.1 (已审核)
> **技术栈**: Rust + Tauri
> **目标平台**: Windows 10/11
> **最后更新**: 2026-02-05
> **审核状态**: ✅ 已通过多角色联合审核

## 一、需求修正说明

### 1.1 原始需求与修正对照

| 编号 | 原始需求 | 修正后需求 | 修正原因 |
|------|----------|-----------|----------|
| 2 | 每周日周期性循环 | 支持周一至周日多选 | 提高灵活性，满足更多场景 |
| 3 | 弹出提示框并倒计时结束后自动锁屏 | 增加"提前通知时间"配置（默认30秒），提示框可延后/立即执行 | 给用户选择权，避免强制中断 |
| 5 | 锁屏、休眠、关机、唤醒 | 一期：锁屏/休眠/关机；**二期：唤醒** | 唤醒需RTC/计划任务支持，复杂度较高 |
| 6 | 设置密码关闭程序 | 明确为"退出程序需密码验证"，增加密码重置机制（密保问题） | 避免密码遗忘导致无法退出 |
| - | 新增 | 配置持久化、开机自启、单实例运行、日志记录 | 基础功能完善 |

### 1.2 关键决策
- ✅ 程序以**管理员权限**运行（支持休眠/关机）
- ✅ **唤醒功能延期至二期**，但保留接口
- ✅ 密码采用 **Argon2id** 哈希存储
- ✅ 配置存储于 `%APPDATA%/TimerApp/config.json`
- ✅ 日志存储于 `%APPDATA%/TimerApp/logs/`

### 1.3 审核后关键修正（v1.1）

| 修正项 | 原设计 | 修正后 | 原因 |
|--------|--------|--------|------|
| 密码使用场景 | 首次启动设密码，含义模糊 | 明确为"退出程序时验证"，启动不需密码 | 避免误解，提升体验 |
| 生效规则逻辑 | 两开关独立，逻辑不明 | 明确为AND关系，需同时满足 | 消除歧义 |
| 提示弹窗策略 | 统一弹窗，不考虑动作类型 | 锁屏=弹窗，休眠/关机=托盘通知 | 避免阻塞自动化 |
| 计时器状态 | 未定义持久化 | 状态持久化，重启后恢复 | 保证连续性 |
| 单实例与弹窗 | 层级关系不明 | 提示弹窗为模态，必须先响应 | 明确交互优先级 |

### 1.4 新增约束条件
- 自定义时间间隔：1-1440 分钟
- 最大连续延后次数：3 次
- 密码错误 3 次锁定 5 分钟
- 日志单文件最大 10MB，总量 100MB

## 二、四阶段实施计划

### 📍 阶段一：MVP核心功能

**目标**: 实现可运行的基础定时器（倒计时 + 锁屏 + 托盘）

**时间**: 第1-2周

| 任务ID | 模块 | 功能点 | 技术方案 | 交付物 | 验证标准 |
|--------|------|--------|----------|--------|----------|
| M1 | 项目搭建 | Tauri+Rust环境配置 | `cargo create-tauri-app` | 可编译运行的空项目 | `cargo tauri dev` 正常启动 |
| M2 | 系统托盘 | 托盘图标、右键菜单 | Tauri `tray` API | 托盘图标常驻 | 1) 最小化到托盘<br>2) 右键菜单有"显示主窗口"、"退出" |
| M3 | 定时器核心 | 倒计时引擎 | Rust `tokio::time` 或 `std::thread` | 精准的倒计时 | 倒计时误差 < 1秒 |
| M4 | 基础UI | 时间设置、倒计时显示 | React/Vue + Tauri Frontend | 美观的界面 | 1) 下拉选择15/30/45/60分钟<br>2) 支持自定义分钟数<br>3) 实时倒计时显示 |
| M5 | 锁屏功能 | 调用Windows锁屏API | `LockWorkStation` via `windows-rs` crate | 自动锁屏执行 | Win10/11测试通过 |
| M6 | 单实例 | 防止多开 | 全局Mutex/Tauri插件 | 单实例运行 | 重复启动激活原窗口 |

**阶段一验收检查清单**:
- [ ] 程序可正常安装并启动
- [ ] 可选择15/30/45/60分钟或自定义时间（1-1440分钟）
- [ ] 倒计时显示准确，误差<1秒
- [ ] 倒计时结束自动锁屏
- [ ] 可最小化到系统托盘
- [ ] 从托盘可恢复主窗口
- [ ] 重复启动时只保持一个实例
- [ ] 计时器状态持久化（运行/暂停）重启后恢复

### A. 配置数据结构 (v1.1)

```json
{
  "version": "1.1",
  "timer": {
    "interval_minutes": 30,
    "custom_interval": false,
    "min_interval": 1,
    "max_interval": 1440,
    "advance_notice_seconds": 30,
    "max_delay_times": 3,
    "delay_options": [5, 10, 30]
  },
  "schedule": {
    "time_limit_enabled": false,
    "weekday_limit_enabled": false,
    "start_time": "09:00",
    "end_time": "18:00",
    "weekdays": [1, 2, 3, 4, 5],
    "logic": "AND"
  },
  "action": {
    "type": "lock",
    "show_notice": true,
    "notice_behavior": {
      "lock": "popup_modal",
      "suspend": "tray_balloon",
      "shutdown": "tray_balloon"
    }
  },
  "startup": {
    "auto_start": false,
    "start_minimized": false,
    "start_timer_automatically": false
  },
  "auth": {
    "enabled": false,
    "password_hash": "",
    "security_question": "",
    "security_answer_hash": "",
    "max_attempts": 3,
    "lockout_duration_seconds": 300
  },
  "state": {
    "timer_status": "idle",
    "remaining_seconds": 0,
    "last_update": "",
    "delay_count": 0
  },
  "log": {
    "level": "info",
    "max_days": 7,
    "max_file_size_mb": 10,
    "max_total_size_mb": 100
  }
}
```

**字段说明**:
- `timer.min_interval/max_interval`: 自定义时间范围限制（1-1440分钟）
- `timer.max_delay_times`: 最大连续延后次数
- `schedule.logic`: 生效规则逻辑，固定为"AND"
- `action.notice_behavior`: 不同动作的通知方式
- `state`: 运行时状态持久化
- `auth.max_attempts/lockout_duration_seconds`: 密码错误限制

### B. 命令行参数

| 参数 | 说明 |
|------|------|
| `--safe-mode` | 安全模式，跳过密码验证 |
| `--minimized` | 启动时最小化到托盘 |
| `--config <path>` | 指定配置文件路径 |
| `--log-level <level>` | 设置日志级别 |
